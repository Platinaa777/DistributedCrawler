# Example 3

Этот пример показывает одноразовый обход каталога MacBook Pro 14 на `biggeek.ru` с переходом в карточки товаров.

## Что лежит в папке

- `request3.json` - конфигурация задания.
- `example-response3.json` - пример результата.
- `items-list.html` - HTML листинга.
- `item-info.html` - HTML карточки товара.

## Что показывает запрос

`request3.json` описывает сценарий, в котором crawler:

- стартует со страницы каталога `biggeek.ru/catalog/macbook-pro-14?...`;
- работает в режиме `CRAWL_MODE_PAGINATION_AND_LINKS`;
- идет по пагинации каталога;
- переходит по найденным товарным ссылкам;
- собирает на листинге item-поля:
  - `name`, `price`, `original_price`, `url`, `thumbnail`;
- собирает на карточке товара page-level fields:
  - `name`, `price`, `original_price`, `sku`, `images`,
  - `characteristic_labels`, `characteristic_values`.

Это пример для интернет-магазина, где на листинге есть краткая выдача, а на product page лежат подробные характеристики товара.

## Что генерируется в ответ

`example-response3.json` показывает:

- массив `results` с записями как по страницам каталога, так и по карточкам;
- на страницах каталога полезные данные находятся в `items`;
- на карточках товара полезные данные находятся в `fields`;
- `total_tasks` показывает общий масштаб обхода по пагинации и ссылкам.

## Что важно для спеки

- При `PAGINATION_AND_LINKS` количество задач сильно больше числа seed URL.
- На разных страницах структура результата может различаться по заполненности, но shape объекта остается единым.
- Для e-commerce сценариев пары `characteristic_labels` и `characteristic_values` можно интерпретировать как набор характеристик товара.
