# Example 5

Этот пример показывает scheduled-задачу: тот же сбор книг, что и в Example 1, но запускаемый по cron.

## Что лежит в папке

- `request5.json` - конфигурация периодического задания.
- `response5.json` - пример результата одного из запусков.
- `All products _ Books to Scrape - Sandbox.html` - HTML листинга.

## Что показывает запрос

`request5.json` почти повторяет структуру `request1.json`, но ключевое отличие такое:

- `job_type` = `JOB_TYPE_SCHEDULED`;
- в `schedule.cron` задано выражение `* * * * *`.

То есть запрос показывает, как оформить не разовый crawl, а регулярное задание, которое должно запускаться по расписанию.

## Что генерируется в ответ

`response5.json` показывает результат одного конкретного выполнения scheduled job:

- структура ответа такая же, как у разового запуска;
- в `results` лежат обработанные страницы каталога;
- каждая страница содержит `fields.page_title` и список `items`;
- `total_tasks` показывает число страниц, обработанных в рамках именно этого запуска.

## Что важно для спеки

- Отличие scheduled job от once job находится в конфигурации запуска, а не в формате результата.
- Ответ scheduled-задачи остается совместимым с ответом разового запуска.
- Для потребителя API это значит, что можно переиспользовать один и тот же downstream-парсер результатов.
