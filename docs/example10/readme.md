# Example 10

Этот пример показывает гибридный обход Wikipedia: стартуем со страницы поиска и переходим в найденные статьи.

## Что лежит в папке

- `request10.json` - конфигурация одноразового задания.
- `readme.md` - описание сценария и ограничений.

## Что показывает запрос

`request10.json` описывает сценарий, в котором crawler:

- стартует с той же search results страницы, что и в Example 9;
- работает в режиме `CRAWL_MODE_PAGINATION_AND_LINKS`;
- ограничен глубиной `max_depth = 1`, чтобы не уходить слишком далеко по wiki-графу;
- разрешает переход только в main article namespace через `allowed_url_patterns`;
- исключает `Main_Page`, чтобы не забирать лишнюю навигационную страницу;
- на search results странице собирает те же item-поля:
  - `title`;
  - `url`;
  - `snippet`;
  - `meta`;
  - `thumbnail`;
- на article pages собирает page-level поля:
  - `title`;
  - `summary`;
  - `image`;
  - `infobox_labels`;
  - `infobox_values`;
  - `categories`.

Это Wikipedia-аналог Examples 2-4: есть listing-страница, а затем detail pages, куда crawler переходит по внутренним ссылкам.

## Почему это хороший шаблон

- Он показывает, как работать со внутренними ссылками Wikipedia, не проваливаясь в `Special:`, `Help:`, `Category:` и другие namespace-ы.
- Его легко адаптировать под другую тему: меняешь поисковую строку и при необходимости корректируешь whitelist.
- Это хороший reference для сценариев "найти статьи по теме и вытащить из каждой summary + infobox".

## Что генерируется в ответ

Ожидаемый ответ по форме совпадает с Examples 2-4:

- в `results` будут и страницы поисковой выдачи, и отдельные статьи;
- у search results страниц полезные данные находятся в `items`;
- у article pages полезные данные находятся в `fields`;
- `total_tasks` покажет суммарное число реально посещенных страниц.

## Что важно для спеки

- Здесь ключевой фильтр - `allowed_url_patterns = ^https://en\\.wikipedia\\.org/wiki/([^:]+)$`, он пропускает только статьи из основного namespace.
- `max_depth = 1` нужен, чтобы crawler переходил из выдачи в статьи, но не начинал бесконечно гулять уже из самих статей.
- Если захочешь повторить сценарий под другую тему, safest way - сначала подобрать query, который точно открывает search results, а не редиректится в один article.
