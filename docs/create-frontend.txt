Prompt for Frontend (Senior Angular + Designer)

You are implementing an MVP frontend for a Distributed Web Crawler system.

Tech stack (mandatory)

Angular v19

Tailwind CSS v4

Angular Material (latest, from https://material.angular.dev/
)

REST via grpc-gateway (OpenAPI / JSON)

Always check official documentation using context7 before implementation decisions

Goal of the UI (MVP)

Provide a minimal but professional UI that allows the user to:

View existing crawl jobs

Open a specific job and inspect its tasks

Create a new crawl job using:

HTML Preview (iframe-based)

DevTools-like element picker

Visual builder for ExtractionSpec (fields + metrics)

Once the user clicks Create Job, everything else happens on the backend.

Backend API (source of truth)

Use the existing grpc-gateway HTTP endpoints:

Jobs

GET /api/v1/jobs

GET /api/v1/jobs/{id}

POST /api/v1/jobs

Tasks

GET /api/v1/jobs/{job_id}/tasks

GET /api/v1/tasks/{id}

Preview (HTML Inspector)

POST /api/v1/previews

GET /api/v1/previews/{id}
→ returns Preview.download_url (presigned MinIO URL)

Pages / Flows to Implement
1) Jobs List Page

Route: /jobs

Fetch jobs via GET /api/v1/jobs

Table (Angular Material mat-table):

Job name

Status

Created at

Click row → navigate to /jobs/:id

Button: Create Job (mat-button / mat-raised-button)

2) Job Details Page

Route: /jobs/:id

Load job via GET /api/v1/jobs/{id}

Load tasks via GET /api/v1/jobs/{job_id}/tasks

Show:

Job config summary (read-only)

Task list (URL, status, depth) using mat-table

Read-only (no editing)

3) Create Job Page (Core MVP)

Route: /jobs/create

This is the most important screen.

Step A — URL Preview

Input: target URL (mat-form-field + mat-input)

Button: Load Preview

Call:

POST /api/v1/previews { url }

then GET /api/v1/previews/{id}

Load preview.download_url

Fetch HTML as text

Render inside:

<iframe
  sandbox="allow-same-origin"
  [srcdoc]="html">
</iframe>


⚠️ No direct external iframe URLs
⚠️ Always use sanitized HTML returned by backend

Step B — DevTools-like Element Picker

Implement a lightweight inspector:

Overlay layer positioned above the iframe

On hover:

highlight DOM element inside iframe

On click:

compute CSS selector

extract:

text content

attributes (href, src, etc.)

Show:

selector

extracted preview value

Use Angular Material components (mat-card, mat-list, mat-chip) for display

This selector is used to build ExtractorSpec.

Step C — ExtractionSpec Builder (Visual DSL)

User builds:

message ExtractionSpec {
  repeated FieldSpec fields
  repeated MetricSpec metrics
}

Fields UI

For each field (Angular Material forms):

name (mat-input)

type (mat-select):

string | int | float | bool | url | json

required (mat-checkbox)

extractor:

source = "html" (MVP only, hidden/fixed)

selector_type = "css" (hidden/fixed)

selector (auto-filled from inspector)

attribute (mat-select: text, href, src, etc.)

multiple (mat-checkbox)

transforms (optional, simple list using mat-select / mat-chip-list)

Metrics UI (minimal)

Support at least (via mat-select):

count

len

word_count

Step D — Job Settings

Simple Angular Material forms:

Job name (mat-input)

Seeds (initial URL, simple list input)

Scope rules:

max_depth (mat-input type="number")

allowed_domains (chip input or textarea)

Rate limit:

rps (mat-input type="number")

Step E — Create Job

Assemble full CrawlJobConfig

Call:

POST /api/v1/jobs

On success:

redirect to /jobs/:id

UX / Design Rules

Minimalist, dev-tool-like

No animations beyond hover / selection

Tailwind for layout & spacing

Angular Material for all UI components

Dark-mode friendly (optional but preferred)

Everything must feel deterministic and inspectable

Non-Goals (Do NOT implement)

No authentication

No real-time updates

No editing jobs after creation

No SPA JavaScript execution inside iframe
