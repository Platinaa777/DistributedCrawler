Ты — технический писатель и backend-инженер. Твоя задача: написать документацию на русском языке для DSL экстракции данных `models.ExtractionSpec` в проекте distributed-crawler.

Контекст:
- Парсинг HTML делается через goquery (CSS-селекторы как в jQuery).
- Экстракция всегда основана на CSS-селекторах (ExtractorSpec.Selector) и атрибуте (ExtractorSpec.Attribute).
- Преобразования (Transforms) применяются последовательно к извлеченному значению.
- Метрики (Metrics) вычисляются на основе извлеченных полей, либо на основе `body_text` / DOM.

Цель документации:
1) Объяснить все параметры `ExtractionSpec` (Fields + Metrics) — что означает каждый параметр и как он работает.
2) Дать много практических примеров JSON-конфигураций (минимум 8–12 штук), включая:
   - извлечение text и атрибутов (href/src/content),
   - multiple + index (включая index=-1),
   - required=true/false и поведение при ошибке,
   - трансформации по цепочке,
   - метрики на основе полей и `body_text`,
   - пример с meta[property='og:url'] → content → upper (как в примере ниже).
3) Отдельно и подробно объяснить CSS-селекторы: 
   - базовые: tag, .class, #id
   - вложенность и пробелы: `div a`, `div > a`, `a + b`, `a ~ b`
   - атрибутные: `[attr]`, `[attr="x"]`, `[attr^="x"]`, `[attr$="x"]`, `[attr*="x"]`, `[attr~="x"]`, `[attr|="x"]`
   - групповая выборка через запятую: `h1, h2`
   - псевдоклассы (которые обычно поддерживаются в селекторах): `:first-child`, `:nth-child(n)` — с оговоркой, что поддержка зависит от движка goquery/cascadia, поэтому если не уверен — так и напиши.
   - обязательно объясни, что означает пробел (descendant) и как читать такие селекторы.
   - приведи примеры на маленьких HTML-фрагментах (вставляй HTML в документацию блоками).

Формат:
- Markdown документ.
- Заголовки и разделы:
  1. Обзор ExtractionSpec
  2. Fields: структура, семантика, жизненный цикл (extract → transforms → type conversion)
  3. ExtractorSpec: selector / attribute / multiple / index — подробно
  4. CSS selectors cheat-sheet (объяснения + мини-примеры HTML + что вернется)
  5. Transforms: полный список операций + что принимает на вход + что возвращает + примеры до/после
  6. ValueType: string/int/float/bool/url/json — как конвертируется и какие ошибки возможны
  7. Metrics: полный список метрик + входные параметры + примеры
  8. Полные примеры JSON конфигов (готовые к копипасту)

Нужно описать следующие сущности (если точные структуры в коде отличаются — адаптируй текст под фактические поля, но смысл оставь):
- ExtractionSpec:
  - fields: []FieldSpec
  - metrics: []MetricSpec

- FieldSpec:
  - name: string (ключ поля в output)
  - label: string (человеческое имя, для UI)
  - type: ValueType (string/int/float/bool/url/json)
  - required: bool (если true и extraction упал — лог warn, но поле все равно может стать nil; опиши это поведение как сейчас реализовано)
  - extractor: ExtractorSpec
  - transforms: []TransformSpec

- ExtractorSpec:
  - selector: string (CSS selector)
  - attribute: string (какой атрибут брать; поддержать "text"/"innerText" и "html"/"innerHTML" как псевдо-атрибуты; объяснить это)
  - multiple: bool (если true — вернуть массив строк, иначе строку)
  - index: *int (если multiple=true и index указан — вернуть один элемент по индексу; если index < 0 — считать с конца, -1 = последний)

- TransformSpec:
  - op: string
  - arg: any (может быть пустым)
Операции transforms, которые нужно описать ВСЕ (каждую отдельно):
  - trim
  - lower
  - upper
  - collapse_ws
  - html_to_text
  - normalize_url
  - unique (для []string)
  - limit (для []string, arg может быть int/float/json.Number/string)
  - parse_int
  - parse_float
  - parse_price (из строки достает число, например "$19.99" -> 19.99)
  - sha256

- MetricSpec:
  - name: string
  - op: string
  - input: string (имя поля, либо специальное "body_text" где применимо)

Метрики (описать ВСЕ):
  - len (длина строки)
  - count (количество непустых элементов в массиве)
  - word_count (если input="body_text" — считать слова по всему bodyText; иначе по строковому значению поля)
  - field_present (true если поле не nil и не пустая строка)
  - count_external_links (считает <a href="http(s)://...">)

Обязательно включи и разбор вот этого примера (в документации прямо вставь его и построчно объясни):
{
  "extraction_spec": {
    "fields": [
      {
        "name": "og_url",
        "type": "string",
        "required": true,
        "label": "OpenGraph URL",
        "extractor": {
          "selector": "meta[property='og:url']",
          "attribute": "content",
          "multiple": false,
          "index": 0
        },
        "transforms": [
          { "op": "upper", "arg": "" }
        ]
      }
    ],
    "metrics": []
  }
}

Требования к качеству:
- Пиши максимально практично: "что написать в selector", "какие типичные ошибки", "как дебажить, если пусто".
- Для multiple+index обязательно покажи пример HTML со списком и пример результата при index=0, index=2, index=-1.
- Для attribute обязательно покажи примеры:
  - attribute="text"
  - attribute="html"
  - attribute="href" (и объясни, что относительные ссылки резолвятся в абсолютные через baseURL)
  - attribute="src"
  - attribute="content" для meta
- Для transforms покажи по 1–2 примера на каждую операцию (до/после).
- В конце добавь "Рекомендации" (best practices): делать селекторы устойчивыми, избегать хрупких nth-child, предпочитать data-* атрибуты если есть, и т.д.
