You are implementing an MVP “HTML Preview for Inspector UI” backend pipeline (Go + gRPC + grpc-gateway + MinIO). Goal: UI can request a sanitized copy of a fetched page HTML and render it inside a sandboxed <iframe srcdoc> (or load it via URL) to support an element picker that outputs CSS selectors.

What to build
1) Public API (gRPC + HTTP via grpc-gateway)

Create a new Preview service with 2 endpoints:

A. Create preview (fetch + store)

Input: url (page URL), optional job_id / task_id if available.

Output: preview_id, minio_key, and either:

download_url (pre-signed GET URL), OR

content_url (an internal gateway URL that streams the HTML)

B. Get preview content

Input: preview_id

Output:

either raw HTML bytes (for UI to set iframe srcdoc)

or a redirect / proxy streaming response

Use grpc-gateway annotations so the UI can call it via HTTP.

Example proto:

syntax = "proto3";

package preview.v1;

option go_package = "distributed-crawler/gen/preview/v1;previewv1";

import "google/api/annotations.proto";
import "google/protobuf/timestamp.proto";

service PreviewService {
  // Fetch remote URL, sanitize HTML, store in MinIO, return preview reference
  rpc CreatePreview(CreatePreviewRequest) returns (CreatePreviewResponse) {
    option (google.api.http) = {
      post: "/v1/previews"
      body: "*"
    };
  }

  // Return sanitized HTML (or metadata) for UI
  rpc GetPreview(GetPreviewRequest) returns (GetPreviewResponse) {
    option (google.api.http) = {
      get: "/v1/previews/{preview_id}"
    };
  }
}

message CreatePreviewRequest {
  string url = 1;
  string job_id = 2;   // optional
  string task_id = 3;  // optional
}

message CreatePreviewResponse {
  string preview_id = 1;
  string minio_key = 2;
  string content_type = 3; // "text/html"
  string download_url = 4; // presigned MinIO URL OR empty
  string content_url  = 5; // gateway URL OR empty
  google.protobuf.Timestamp created_at = 6;
}

message GetPreviewRequest {
  string preview_id = 1;
}

message GetPreviewResponse {
  string preview_id = 1;
  string url = 2;
  string minio_key = 3;
  string content_type = 4;
  string download_url = 5; // optional
  google.protobuf.Timestamp created_at = 6;
}


Notes:

A practical MVP: CreatePreview returns content_url like /v1/previews/{id}:raw (custom HTTP handler) or a pre-signed download_url.

2) Backend pipeline steps (CreatePreview)

Implement CreatePreview(url):

Fetch HTML

Use your existing fetcher (same as FetchWorker), but in MainServer context.

Record final_url, status_code, content_type, body.

Sanitize HTML for safe iframe rendering

Strip <script> tags.

Remove inline event handlers like onclick=, onload=, etc.

Remove dangerous tags: iframe, object, embed.

(Optional) rewrite external resource URLs (img/src, link/href) or leave them; UI iframe sandbox should prevent JS anyway.

Ensure output is still valid HTML and deterministic.

Store sanitized HTML in MinIO

Generate key like: previews/{preview_id}.html (or previews/{job_id}/{task_id}.html).

contentStore.Store(ctx, key, sanitizedHTML, "text/html; charset=utf-8")

Persist preview metadata

Store in DB table (recommended) previews:

preview_id, source_url, final_url, minio_key, created_at, maybe expires_at.

Return preview_id in response.

Return access method
Choose one of two MVP options (implement at least one):

Option 1: Pre-signed MinIO URL

Use MinIO client to generate PresignedGetObject with short TTL (e.g., 10–60 minutes).

Return download_url to UI.

UI loads HTML via fetch and then sets iframe.srcdoc = html.

Recommendation for MVP:

If you already have MinIO presign utilities: Option 1 is simplest.

3) What UI receives / uses

UI calls POST /v1/previews { url }

Backend responds with:

preview_id

download_url OR content_url

UI fetches HTML text and sets:

iframe.srcdoc = sanitized_html

iframe.sandbox = "" (strictest) or "allow-same-origin" if you need DOM access; prefer strict + do inspection from overlay in parent via postMessage if possible, but MVP can use allow-same-origin to read DOM and build CSS selectors.

Also return content_type and created_at for debugging.

Implementation constraints

Keep it minimal and production-minded: timeouts on fetch, max HTML size limit, and short presign TTL.

Do NOT execute third-party JS: sanitize HTML + sandbox iframe.

Do not store raw cookies; fetch as an unauthenticated request unless job auth is explicitly provided.

Deliverable:

preview.v1 proto + server implementation in Go

MinIO store + (optional) presign or streaming download endpoint

DB model/repo for preview metadata
