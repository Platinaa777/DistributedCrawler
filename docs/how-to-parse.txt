Вариант A (рекомендую для MVP): показывать свою копию HTML в sandbox-iframe

Как это работает:

Твой backend (FetchWorker) скачал страницу и сохранил raw HTML (в MinIO).

В UI ты запрашиваешь у своего backend: “дай HTML этой страницы” (или уже подготовленный “preview html”).

UI вставляет HTML в <iframe srcdoc="..."> (или в iframe через blob URL).

Важно: вырубить скрипты и опасные вещи:

вырезать <script>, inline handlers onclick=...

запретить iframe, object, etc

поставить sandbox на iframe (например sandbox="allow-same-origin" или даже без него, в зависимости от реализации)

Ты накладываешь поверх iframe режим “выбор элемента”: пользователь кликает по заголовку/цене → ты получаешь CSS селектор и сразу показываешь превью.

Почему это удобно:

Всё same-origin (HTML приходит от твоего домена), значит ты можешь читать DOM внутри iframe и строить селекторы.

Нет CORS/CSP проблем, нет утечки cookies, нет выполнения JS конкурента.

Минус:

Если сайт сильно JS-овый (SPA), “голый” HTML может быть пустым.
