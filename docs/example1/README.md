# Example 1

Этот пример показывает одноразовый пагинируемый сбор каталога книг с листинга `books.toscrape.com`.

## Что лежит в папке

- `request1.json` - конфигурация задания на одноразовый запуск.
- `example-response.json` - пример результата выполнения этого задания.
- `All products _ Books to Scrape - Sandbox.html` - локальный HTML листинга, по которому подбирались селекторы.

## Что показывает запрос

`request1.json` описывает сценарий, в котором crawler:

- стартует с `https://books.toscrape.com/`;
- работает в режиме `CRAWL_MODE_PAGINATION_ONLY`;
- ходит только по домену `books.toscrape.com`;
- идет по пагинации через `ul.pager li.next a`;
- извлекает поля страницы `page_title`;
- извлекает из каждого товара `name`, `price`, `availability`, `url`.

Это базовый пример для сценария "собрать все карточки со страниц листинга без перехода в карточку товара".

## Что генерируется в ответ

`example-response.json` показывает, что результат содержит:

- `job_id` и `exported_at` - метаданные выгрузки;
- `results` - массив обработанных страниц;
- для каждой страницы:
  - `task_id`, `url`, `status`;
  - `fields.page_title` - общие поля страницы;
  - `items` - массив товаров, найденных на этой странице;
- `total_tasks` - количество обработанных страниц пагинации.

## Что важно для спеки

- Один `result` соответствует одной странице каталога.
- Общие данные страницы лежат в `fields`.
- Повторяющиеся сущности каталога лежат в `items`.
- Так как режим `PAGINATION_ONLY`, crawler не уходит в карточки товаров и не собирает дополнительные поля из product page.
