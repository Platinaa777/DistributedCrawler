Implement “link discovery” in ParserWorker: after parsing a page, if the current task depth is below job config Scopes.MaxDepth, discover outgoing links from the HTML and enqueue them as new crawl tasks for the fetcher via Transactional Outbox (do NOT publish to RMQ directly from the parser).

Requirements:

Read current task from DB to get URL, Depth, JobID. Load job + config to get Scopes (MaxDepth, allowed domains/rules).

If task.Depth >= config.Scopes.MaxDepth: stop (no new tasks).

Otherwise:

Extract links from HTML: all <a href>.

Normalize and resolve relative URLs using the page FinalURL (or URL if FinalURL is empty).

Filter:

only http/https

pass scopeValidator.Validate(url, nextDepth, config.Scopes)

dedupe (within page), and optionally skip already-existing tasks (repo check or unique constraint).

For each accepted link:

create CrawlTask{ ID, JobID, URL, Depth: task.Depth+1, Status: Pending, EnqueuedAt: now }

create an outbox event like TaskEnqueuedEvent(taskID, jobID, url, enqueuedAt) and store it in outbox table.

Persist tasks + outbox events atomically:

either taskRepo.BulkCreate(...) + outboxRepo.BulkCreate(...) inside the same DB transaction,

or taskRepo.Create(...) + outboxRepo.Create(...) per item but still within one transaction.

Another worker (existing outbox publisher) will pick outbox events and publish them to the crawl/fetch queue.

Keep parser output behavior unchanged; just add the “enqueue next URLs” step at the end of handleMessage() after successful extraction.

Edge cases:

Avoid mailto:, javascript:, fragments-only links.

Respect MaxDepth strictly (nextDepth = currentDepth + 1).

Use idempotency: rely on DB uniqueness (recommended: unique index on (job_id, url) or (job_id, url_hash)) and ignore conflicts.

Deliverables:

ParserWorker: new function discoverAndEnqueueLinks(ctx, crawlTask, htmlContent, config) that does extraction + filtering + transactional task/outbox creation.

Add/extend repos/interfaces for OutboxRepository and possibly BulkCreate + transaction boundary support.
