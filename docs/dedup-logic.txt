You are implementing deduplication for crawl tasks in a Go backend for a distributed crawler.

Domain model

A CrawlJob contains many CrawlTask.

Each Crawl Task parse HTML page, we need deduplication logic. If for crawl_job exists crawl_task which parsed html page with some
hash, in the future we should skip it (to avoid circles)

Goal

Before creating a new crawl_task, check whether a task with the same hash already exists for the same crawl_job.
If it exists → skip (do nothing, no DB insert, no outbox event, no RMQ message).

Requirements
1) Hash definition (deterministic)

crawl task already has BodyHash, you should check it while exporting to S3 for future crawl_task, if in DB
this value exists, we should complete crawl_task without do anything (in another crawl_task it was done)

type CrawlTask struct {
	ID valueobjects.CrawlTaskID

	JobID valueobjects.CrawlJobID
	Job   *CrawlJob

	URL        string
	FinalURL   *string // After redirects
	Status     TaskStatus
	EnqueuedAt time.Time

	Depth          uint64
	BodyHash       string // SHA-256
	MinioObjectKey string

	// Result persistence fields (Part A - ParserWorker)
	ResultObjectKey   *string    // S3 object key for result JSON
	ResultContentType *string    // Content type (e.g., "application/json")
	ResultSizeBytes   *int64     // Size of result file
	ResultCreatedAt   *time.Time // When result was stored
}

2) Database-level protection (strongly preferred)

Enforce uniqueness in DB to avoid races between workers.

Add a migration:

Add column hash to crawl_tasks table (if not exists) or confirm it exists

Create a unique index:

UNIQUE (crawl_job_id, hash)

If tasks are soft-deleted or have statuses, still keep uniqueness for MVP.

3) Repository API

Extend CrawlTaskRepo with methods:

ExistsByJobIDAndHash(ctx, jobID, bodyHash) (bool, error)

Alternatively: GetByJobIDAndHash returning task or nil

Also add an insert that can handle DB uniqueness gracefully:

If insert fails with unique-violation on (crawl_job_id, hash), treat as dedup hit (skip) and return a typed error like ErrTaskAlreadyExists.
