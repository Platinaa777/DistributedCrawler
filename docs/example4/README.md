# Example 4

Этот пример показывает тот же BigGeek-сценарий, что и Example 3, но с дополнительным белым списком URL для переходов в карточки.

## Что лежит в папке

- `request4.json` - конфигурация задания.
- `example-response4.json` - пример результата.
- `items-list.html` - HTML листинга.
- `item-info.html` - HTML карточки товара.

## Что показывает запрос

`request4.json` отличается от предыдущего примера тем, что в `scopes` добавлен `allowed_url_patterns`:

- разрешены только ссылки вида `https://biggeek.ru/products/noutbuk-apple-macbook-pro-14-*`;
- остальные ссылки, даже если они встречаются в каталоге, не должны попадать в обход;
- при этом сохраняются пагинация, сбор товаров с листинга и сбор характеристик из product page.

Это пример для спеки на тему "как ограничить линковый обход только нужным типом карточек".

## Что генерируется в ответ

`example-response4.json` имеет ту же общую форму, что и в Example 3:

- `results` содержит и страницы каталога, и карточки товаров;
- у листингов полезный результат лежит в `items`;
- у карточек полезный результат лежит в `fields`;
- `total_tasks` отражает только реально разрешенные к обходу страницы.

## Что важно для спеки

- `allowed_url_patterns` работает как дополнительный фильтр поверх домена и глубины.
- Такой режим защищает от "утечки" обхода в нерелевантные карточки, аксессуары и соседние разделы.
- Это рекомендуемый шаблон для каталогов, где на листинге много побочных ссылок.
